Android Studio的环境搭建

  • Andriod Studio
  • 安装配置

背景

Android Studio 是这个学期的选修课《移动应用测试》所需要的软件之一，因此需要自己进行安装，而在安装的过程中也是遇到了很多麻烦。这篇博客就是为了总结一下处理方法而诞生的。

安装过程

1. 安装JDK

因为在去年的小学期我们学习的就是java编程，所以这台电脑上已经装好了jdk，这一步先略去。

2. 安装SDK

SDK工具使用的是东软的镜像网站所提供的[build-tools_r25.2.5-windows.zip]（地址在这里，可能不同地区的用户下载速度也会不同，网上有一部分人用的是郑州大学的镜像）。下载解压之后进入tools文件夹，运行android.bat文件就会弹出SDK Manager，由于国内大概率访问不到dl.google.com，packages基本是刷新不出来的，所以这里有两种解决方案：

  1. 使用镜像源

    ​ 我使用的是老师提供的东软镜像源，打开菜单栏中Tools->options，把HTTP Proxy Server改为mirrors.neusoft.edu.cn，下面的Port改为80，并勾选Force https://... sources to be fetched using http://...，然后重新打开SDK Manager，然后就可以看到刷新出的文件了。

    ​ 但！是！东软的镜像网站提供的是不完全的版本，在进行之后的设置时仍然会遇到问题，所以我更建议使用第2种方法。

  2. 改hosts文件

    ​ 在网上找一个ip查询工具查一下dl.google.com的ip，在本机ping一次确定可用，把这个ip写到hosts中，完成。

在SDK Manager中需要安装的部件有：

  • Platform
  • Platform-tools
  • Build-tools
  • tools

第一个可以直接选择一个Android版本全部安装下来（可能会有不需要的文件），后三个全部都是列表中最前面的几个，Build-tools需要之后选择对应的版本

3.安装Android Studio

可以直接去官网下载，或者在Android Studio中文组官网进行下载

下载安装之后，因为我们在上一步已经下载好了SDK，所以在AS的安装目录中打开bin->idea.properties，在这个文件最后一行键入：

  • disable.android.first.run=true

4. 初次启动Android Studio

配置完成后打开Android Studio，在选择组件时去掉SDK，选择Virtual Device（如果有的话）
