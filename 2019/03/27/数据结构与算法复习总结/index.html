绪论

  • 数据的逻辑结构由数据结点(Node)和连接两个结点的边二元关系组成

结点的数据类型

  • 整数类型
  • 实数类型
  • 布尔类型
  • 字符类型
  • 指针类型
  • 其它复合类型

结构的分类

  • 线性结构：每个结点在关系r上最多只有一个前驱结点和一个后继结点
  • 树形结构：除了根结点以外，所有结点均有且只有唯一的前驱结点
  • 图结构（网络结构）：对结点的前驱或后继数目无约束

数据的存储结构

  • 顺序方法：把一组结点存放在一片地址相邻的存储单元中。

  • 链接方法：在结点的存储结构中附加指针域来存储结点间的逻辑关系。

  • 索引方法：通过建造一个由数据域Z映射到存储地址域D是索引函数，形成一个存储了一组指针的索引表，每个指针指向存储区域的一个数据结点。

    • 提高了检索效率
  • 散列方法：利用散列函数进行索引值的计算，然后通过索引表求出结点的指针地址。

算法性质

  • 通用性
  • 有效性
  • 确定性
  • 有穷性

算法分析：大O表示法

给出了函数增长率的上限

O(1) < O(log(n)) < O(n) < O(nlog(n)) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n)

线性表

存储结构主要有两类：定长的顺序存储结构(顺序表)和变长的线性存储结构(链表)

顺序表

时间复杂度

  1. 插入：O(n)
  2. 删除：O(n)
  3. 按位置读取元素：O(1)
  4. 按内容查找：O(n)

局限

  • 长度无法改变
  • 插入和删除耗时长

链表

单链表

  • 使用带头结点的单链表有以下好处：
    • 对首结点的插入和删除不需要做特殊处理
    • 对空链表不需要特殊处理

时间复杂度

  1. 检索：O(n)
  2. 插入和删除：本身只需O(1)，但是需要O(n)的定位操作

局限

指针域仅指向后继结点，不能有效找到其前驱结点

双链表

循环链表

从循环链表的任一点出发，都能访问到其他结点

线性表的比较

  • 顺序表：
    • 存储利用率高
    • 存取操作方便
  • 链表
    • 无需事先定义最大长度
    • 插入和删除操作方便

栈LIFO

所有操作限定在线性表的一端进行，每次取出的是最后进栈的元素。

时间复杂度

  1. 插入和删除：O(1)

队列FIFO

元素只能从表的一端插入，从表的另一端删除

时间复杂度

  • 当使用顺序队列时，出队与入队其中一个为O(n)，一个为O(1)

  • 使用循环队列时，均为O(1)

字符串

KMP算法

树

基本性质

  1. 结点数等于所有结点的度数+1
  2. 度为m 的树，其第i 层上至多有m^i 个结点（根结点为第0层）
  3. 高度为h ，度为m 的树至多有 (m^h - 1)/(m - 1)个结点
  4. 具有n 个结点的度为m 的树，其最小高度为（向上取整）log_m(n(m-1)+1)

二叉树

  • 定义：所有结点的度小于等于2的树称为二叉树

  • 二叉树是有序树（左右子树的次序不可颠倒）

  • 特殊的二叉树

    • 完全二叉树：从上到下从左到右依次填充
    • 满二叉树：每一层都填充满
    • 扩充二叉树：把原二叉树的所有结点中出现空子树的位置上都增加特殊的空树叶得到的二叉树
      • 外部结点（新增加的空树叶）的数量等于内部结点（原来二叉树中的结点）的数量加1
      • 外部路径长度（从根结点到每个外部结点的路径长度之和）等于内部路径长度加两倍的内部结点数量

性质

  • 度为0的结点比度为2的结点多一个
  • 第i 层上至多有2^i 个结点
  • 高度为h 的二叉树至多有 2^h - 1个结点
  • 非空满二叉树的叶子结点的数量等于分支结点的数量加1
  • 对一个有n 个结点的完全二叉树的结点按层从0编号，则结点i的双亲结点的编号为(i-1)/2，左子结点的编号为 2i+1，右子结点的编号为 2i+2

遍历

  • 广度优先遍历（子结点依次进队，然后访问出队的结点）
  • 深度优先遍历：
    • 前序遍历（访问该结点，把右子树的根入栈，遍历左子树，无左子树时弹栈）
    • 中序遍历（把该结点压栈，遍历左子树，无左子树时弹栈并访问，遍历右子树）
    • 后序遍历（左子结点依次压栈，读取栈顶元素，访问右子树，右子树被访问过时弹栈访问该结点）

给定前序中序或者中序后序遍历序列可以唯一确定二叉树

二叉搜索树

定义：左子树的关键码小于本身的关键码小于右子树的关键码

中序遍历二叉搜索树可以得到从小到大排序的关键码序列

删除操作

  1. 无子树：直接删除
  2. 有一棵子树：用子树的根结点替换
  3. 有两棵子树：合并删除（用左子树的最右边接右子树的根）或复制删除（左子树最右边的点替换删除的点，将其左子树变为它的父节点的右子树）

平衡二叉树

检索插入删除的时间复杂度均为：O(log_2(n)）

定义：左子树和右子树的高度差不超过1

插入

左单旋、右单旋、左右双旋、右左双旋

删除

核心：子树高度不改变时才停止回溯

对于一个有n个结点的avl树，高度不超过1.5log_2 (n+1)

堆

定义：每个结点的值都大于（小于）其子结点的值的完全二叉树

插入和删除是O(log(n))的

构建：筛选法

O(n)

先所有关键码放入一颗完全二叉树中，之后从第一个具有孩子的结点n/2-1（向下取整）开始，依次调整以它为根的子树，直到根结点被调整结束。

优先队列*

能被删除和访问的是具有最高优先级的元素，可以通过最大堆来实现

huffman编码树

又称最优二叉树，是加权路径长度最短的二叉树

构建方法

从结点集合中选取两棵权值最小的树作为左右子树来构造一棵新的二叉树，且新的二叉树的权值为左右子树根结点权值之和。

将这两棵树删除，把新的二叉树加入F中

重复以上步骤直至集合中只有一棵树

图

常用图的基本类型：无向图，有向图，带权图

生成树

对于连通图G，若连通子图G’包含G中所有顶点和一部分边，且不形成回路，则G’是G的生成树。

生成树不唯一

图的存储

  • 邻接矩阵
  • 邻接表/逆邻接表
  • 十字链表
  • 邻接多重表

图的遍历

深度优先搜索（DFS）

使用栈实现

时间复杂度

对于邻接矩阵所需时间为O(n^2)，对于邻接表的复杂度为O(n+e)

广度优先搜索（BFS）

使用队列实现

时间复杂度

与DFS相同

最小生成树

对于带权无向图，使得各边的权重之和最小的生成树

prim算法

寻找一端已在集合中而另一端不在集合中的边的最小边，并将其加入集合

时间复杂度

O(n^2)，与图中的边数无关，因此适合稠密图

kruskal算法

确定权值最小的边（最小堆）；判断边的两个端点是否在同一个连通分量中（并查集）；不是，则合并两个连同分量。

时间复杂度

向堆中插入了e条边：O(elog(e))

从堆中最多删除e条边：O(elog(e))

最多2e次查找操作：O(e)

最多e次合并操作：O(e)

所以总时间复杂度是O(elog(e))，适用于构造稀疏图的最小生成树

最短路径

单源最短路：Dijkstra算法

按照路径长度递增的次序产生到各顶点最短路径的贪心算法

从dis数组中选取最小值，将对应端点加入S中；从这个端点出发且终点不在S中的边中选出最小的边，更新最短路；重复直至S包含了所有端点

时间复杂度

选择最短dis时如果用简单选择法，复杂度为O(n^2)；如果用最小堆，复杂度为O(nlog(n))
