数据结构与算法复习总结

  • 学习笔记
  • 算法

本人对于大学期间《数据结构与算法》课程的所做复习时的一些总结。

已完成

绪论

  • 数据的逻辑结构由数据结点(Node)和连接两个结点的边二元关系组成

结点的数据类型

  • 整数类型
  • 实数类型
  • 布尔类型
  • 字符类型
  • 指针类型
  • 其它复合类型

结构的分类

  • 线性结构：每个结点在关系r上最多只有一个前驱结点和一个后继结点
  • 树形结构：除了根结点以外，所有结点均有且只有唯一的前驱结点
  • 图结构（网络结构）：对结点的前驱或后继数目无约束

数据的存储结构

  • 顺序方法：把一组结点存放在一片地址相邻的存储单元中。

  • 链接方法：在结点的存储结构中附加指针域来存储结点间的逻辑关系。

  • 索引方法：通过建造一个由数据域Z映射到存储地址域D是索引函数，形成一个存储了一组指针的索引表，每个指针指向存储区域的一个数据结点。

    • 提高了检索效率
  • 散列方法：利用散列函数进行索引值的计算，然后通过索引表求出结点的指针地址。

算法性质

  • 通用性
  • 有效性
  • 确定性
  • 有穷性

算法分析：大O表示法

给出了函数增长率的上限

O(1) < O(log(n)) < O(n) < O(nlog(n)) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n)

线性表

存储结构主要有两类：定长的顺序存储结构(顺序表)和变长的线性存储结构(链表)

顺序表

时间复杂度

  1. 插入：O(n)
  2. 删除：O(n)
  3. 按位置读取元素：O(1)
  4. 按内容查找：O(n)

局限

  • 长度无法改变
  • 插入和删除耗时长

链表

单链表

  • 使用带头结点的单链表有以下好处：
    • 对首结点的插入和删除不需要做特殊处理
    • 对空链表不需要特殊处理

时间复杂度

  1. 检索：O(n)
  2. 插入和删除：本身只需O(1)，但是需要O(n)的定位操作

局限

指针域仅指向后继结点，不能有效找到其前驱结点

双链表

循环链表

从循环链表的任一点出发，都能访问到其他结点

线性表的比较

  • 顺序表：
    • 存储利用率高
    • 存取操作方便
  • 链表
    • 无需事先定义最大长度
    • 插入和删除操作方便

栈LIFO

所有操作限定在线性表的一端进行，每次取出的是最后进栈的元素。

时间复杂度

  1. 插入和删除：O(1)

队列FIFO

元素只能从表的一端插入，从表的另一端删除

时间复杂度

  • 当使用顺序队列时，出队与入队其中一个为O(n)，一个为O(1)

  • 使用循环队列时，均为O(1)

字符串

KMP算法

树

基本性质

  1. 结点数等于所有结点的度数+1
  2. 度为m 的树，其第i 层上至多有m^i 个结点（根结点为第0层）
  3. 高度为h ，度为m 的树至多有 (m^h - 1)/(m - 1)个结点
  4. 具有n 个结点的度为m 的树，其最小高度为（向上取整）log_m(n(m-1)+1)

二叉树

  • 定义：所有结点的度小于等于2的树称为二叉树

  • 二叉树是有序树（左右子树的次序不可颠倒）

  • 特殊的二叉树

    • 完全二叉树：从上到下从左到右依次填充
    • 满二叉树：每一层都填充满
    • 扩充二叉树：把原二叉树的所有结点中出现空子树的位置上都增加特殊的空树叶得到的二叉树
      • 外部结点（新增加的空树叶）的数量等于内部结点（原来二叉树中的结点）的数量加1
      • 外部路径长度（从根结点到每个外部结点的路径长度之和）等于内部路径长度加两倍的内部结点数量

性质

  • 度为0的结点比度为2的结点多一个
  • 第i 层上至多有2^i 个结点
  • 高度为h 的二叉树至多有 2^h - 1个结点
  • 非空满二叉树的叶子结点的数量等于分支结点的数量加1
  • 对一个有n 个结点的完全二叉树的结点按层从0编号，则结点i的双亲结点的编号为(i-1)/2，左子结点的编号为 2i+1，右子结点的编号为 2i+2

遍历

  • 广度优先遍历（子结点依次进队，然后访问出队的结点）
  • 深度优先遍历：
    • 前序遍历（访问该结点，把右子树的根入栈，遍历左子树，无左子树时弹栈）
    • 中序遍历（把该结点压栈，遍历左子树，无左子树时弹栈并访问，遍历右子树）
    • 后序遍历（左子结点依次压栈，读取栈顶元素，访问右子树，右子树被访问过时弹栈访问该结点）

给定前序中序或者中序后序遍历序列可以唯一确定二叉树

二叉搜索树

定义：左子树的关键码小于本身的关键码小于右子树的关键码

中序遍历二叉搜索树可以得到从小到大排序的关键码序列

删除操作

  1. 无子树：直接删除
  2. 有一棵子树：用子树的根结点替换
  3. 有两棵子树：合并删除（用左子树的最右边接右子树的根）或复制删除（左子树最右边的点替换删除的点，将其左子树变为它的父节点的右子树）

平衡二叉树

检索插入删除的时间复杂度均为：O(log_2(n)）

定义：左子树和右子树的高度差不超过1

插入

左单旋、右单旋、左右双旋、右左双旋

删除

核心：子树高度不改变时才停止回溯

对于一个有n个结点的avl树，高度不超过1.5log_2 (n+1)

堆

定义：每个结点的值都大于（小于）其子结点的值的完全二叉树

插入和删除是O(log(n))的

构建：筛选法

O(n)

先所有关键码放入一颗完全二叉树中，之后从第一个具有孩子的结点n/2-1（向下取整）开始，依次调整以它为根的子树，直到根结点被调整结束。

优先队列*

能被删除和访问的是具有最高优先级的元素，可以通过最大堆来实现

huffman编码树

又称最优二叉树，是加权路径长度最短的二叉树

构建方法

从结点集合中选取两棵权值最小的树作为左右子树来构造一棵新的二叉树，且新的二叉树的权值为左右子树根结点权值之和。

将这两棵树删除，把新的二叉树加入F中

重复以上步骤直至集合中只有一棵树

图

常用图的基本类型：无向图，有向图，带权图

生成树

对于连通图G，若连通子图G’包含G中所有顶点和一部分边，且不形成回路，则G’是G的生成树。

生成树不唯一

图的存储

  • 邻接矩阵
  • 邻接表/逆邻接表
  • 十字链表
  • 邻接多重表

图的遍历

深度优先搜索（DFS）

使用栈实现

时间复杂度

对于邻接矩阵所需时间为O(n^2)，对于邻接表的复杂度为O(n+e)

广度优先搜索（BFS）

使用队列实现

时间复杂度

与DFS相同

最小生成树

对于带权无向图，使得各边的权重之和最小的生成树

prim算法

寻找一端已在集合中而另一端不在集合中的边的最小边，并将其加入集合

时间复杂度

O(n^2)，与图中的边数无关，因此适合稠密图

kruskal算法

确定权值最小的边（最小堆）；判断边的两个端点是否在同一个连通分量中（并查集）；不是，则合并两个连同分量。

时间复杂度

向堆中插入了e条边：O(elog(e))

从堆中最多删除e条边：O(elog(e))

最多2e次查找操作：O(e)

最多e次合并操作：O(e)

所以总时间复杂度是O(elog(e))，适用于构造稀疏图的最小生成树

最短路径

单源最短路：Dijkstra算法

按照路径长度递增的次序产生到各顶点最短路径的贪心算法

从dis数组中选取最小值，将对应端点加入S中；从这个端点出发且终点不在S中的边中选出最小的边，更新最短路；重复直至S包含了所有端点

时间复杂度

选择最短dis时如果用简单选择法，复杂度为O(n^2)；如果用最小堆，复杂度为O(nlog(n))

顶点对之间的最短路：Floyd算法

动态规划算法，时间复杂度仍为O(n^3)，但是思路更简单

定义adj矩阵与path矩阵，adj矩阵初始化为所有边（仅计算出），path矩阵为顶点自身；遍历n次矩阵，每次遍历adj矩阵，当adj[i][k] == INF时直接进行下一次循环（因为点i到点k不可达，一定不能更新矩阵）；否则，当 adj[i][k] + adj[k][j] < adj[i][j] 时更新adj[i][j] = adj[i][k] + adj[k][j] 以及 path[i][j] = k。

遍历完成后adj[i][j] 即为i到j的最短路径，path[i][j] 就是i到j的最短路径中j之前的结点，只需循环即可找出最短路的所有结点。

拓扑排序

无向图中是否存在环可以直接用深度优先搜索查找；而有向图中则需要使用拓扑排序。

算法思想

找出任意一个没有入边的顶点，然后输出该顶点，并从图中删除该顶点和由它指出的边，修正其余顶点的入度信息。当图中找不到没有入边的顶点时，若所有的顶点都访问过，则图中无环。

时间复杂度

通过维护一个入度为0的顶点的列表来进行简化(Kahn算法)；初始化入度为0的列表的时间复杂度为O(n+e)，读取列表中的每一个结点并遍历所有边的复杂度同样为O(n+e)，所以总的时间复杂度为O(n+e)；

逆拓扑排序

与拓扑排序相反，寻找出度为0的顶点。

关键路径

AOE网中只有一个入度为0的点（源点）和一个初度为0的点（汇点）。可以用来估算项目的完成时间（关键路径）

先用拓扑排序计算各个顶点的最早发生时间，然后用逆拓扑排序计算顶点的最晚发生时间；如果边的前置顶点的最早发生时间与边的后继顶点的最晚发生时间减去权重相等的话，这条边就属于关键路径

时间复杂度

使用邻接表时复杂度为O(n+e)

查找

静态查找

指在查找过程中不更改数据集中的数据

1. 顺序查找

等概率下查找成功的平均查找长度为(n+1)/2；查找失败需要比较(n+1)次

优点：

对表的特性没有要求；插入元素可以直接加到表尾

缺点：

平均查找长度大，平均和最坏情况下时间复杂度都是O(n)，当数据规模较大时，查找效率低

2. 折半查找（二分查找）

需要数据有序地存储在线性表中。（因为查找中需要对数据元素通过编号进行随机访问）

判定树

树中每个结点用于表示算法中参与比较的数据元素编号的二叉树

  1. 具有n个结点的判定树的高度为log_2(n) + 1（向下取整）

  • 折半查找法查找成功的平均查找长度为((n+1)/n)log_2(n+1) - 1；最坏情况下或查找不成功均为log_2(n) + 1；

  • 只适用于顺序存储的有序表；新增或删除数据时操作复杂

3. 分块查找（索引顺序查找）

将n个数据元素划分为m块，块中的数据不必有序，但块与块之间必须“按块有序”，即第i块中的任意数据都必须小于第i+1块中的任意数据。对每块建立一个索引项，形成具有m个索引项的索引表。索引项包括块中的最大关键字和第一个记录的位置。索引表中的各个索引项按关键字有序。

查找时索引表折半查找确定数据块，然后在数据块中顺序查找

分块查找的平均查找长度：((n+1)/n) log_2(n/m + 1) + (m+1)/2

动态查找

主要通过树结构来实现

B-树（B树）

是一种平衡的多路查找树。一棵m阶的B树，或为空树，或满足以下性质：

  • 树中的每个结点最多有m棵子树
  • 根结点至少有两棵子树
  • 除根结点之外的所有非叶结点至少有m/2 棵子树
  • 所有叶子结点都出现在同一层
  • 有n棵子树的非叶结点包含了n-1个关键字：(n, T0,K1,T1,K2,…,Kn,Tn)
    • m/2 - 1 <= n <= m-1，为关键字的个数，n+1为子树个数
    • Ki为关键字且从小到大排序
    • Ti为指向子树根结点的指针，且指针T(i-1)所指子树的所有结点的关键字均小于Ki，均大于K(i-1)

B-树至少是半满的、有较少的层，而且是完全平衡的

性质：

  • 含有n个结点的m阶B-树的最大高度不超过log_m/2((N+1)/2) + 1

插入：

向m阶B-树的叶结点中添加关键字，若该结点中关键字已满，则将叶结点分裂成两个，并把一个关键字提升到双亲结点中，重复此过程直到某个结点的关键字未满或者产生新的根结点。

删除：

  • 删除的结点不是叶子结点，则用左子树的最大结点（或者右子树的最小结点）替换该结点并删除该结点
  • 结点删除后的关键字个数不小于m/2-1，结束
  • 兄弟结点有多余关键字时，用左兄弟的最右侧关键字（或右兄弟的最左侧关键字）替换双亲结点中划分这两个结点的关键字，将双亲结点中被替换的关键字加入此节点中
  • 兄弟结点都没有多余关键字时，将p，p的左（右）兄弟结点以及双亲结点中划分这两个结点的关键字合并。重复以上步骤

B+树

B-树的变形树，常用于数据库和文件系统中。

优点：

  • 能够保持数据稳定有序
  • 插入和修改有稳定的时间复杂度

一个m阶的B+树或为空树，满足以下特点：

  • 树中的每个结点最多有m棵子树
  • 根结点至少有两棵子树
  • 除根结点之外的所有非叶结点至少有m/2 棵子树
  • 所有叶子结点都出现在同一层
  • 有n棵子树的非叶子结点包含n个关键字：(n,T1,K1,T2,K2,…,Tn,Kn)
    • 其中n为关键字的个数，即子树个数
    • Ki为关键字，从小到大排序
    • Ti为指向子树根结点的指针，子树中所有结点的关键字均不大于Ki
  • 所有子树连接成一个序列

查找

在查找时，无论非叶子结点上的关键字是否与给定关键字相同，都要向下查找到叶子结点。

插入

当结点中的关键字个数大于m时要分裂成两个结点，把划分两个结点的关键字复制到双亲结点中

删除

当叶结点的最大关键字被删除时，其在非叶结点中的副本仍可作为一个分界关键字存在。若删除使叶子结点中关键字的个数小于m/2时，如果该叶结点的兄弟结点关键字大于m/2，则将关键字重新分配。否则将其合并并删除父节点中的关键字、重复以上步骤

散列

在记录的关键字与它的存储位置之间建立一个确定的对应函数关系，使得每个关键字与结构中一个唯一的存储位置相对应。在查找时，首先对记录的关键字进行函数计算，把函数值当做该记录的存储位置，在结构中按此位置取记录比较。

散列函数

  1. 散列函数的定义域必须包括需要存储的全部关键字，如果散列表有m个地址时，其值域必须在0~m-1之间
  2. 散列函数计算出的地址应能均匀分布在整个地址空间中
  3. 应能在短时间内计算出结果

直接定址法

Hash(key) = a*key + b

数字分析法

设有n个d位数，每一位可能有r种不同的符号。可根据散列表的大小选取其中各种符号分布均匀的若干位作为散列地址

这种方法完全依赖于关键码集合

除留余数法

Hash(key) = key%p

平方取中法

先计算关键字的平方值，然后根据散列表长度取中间的几位数作为散列函数值

基数转换法

将关键字转换为另一种进制的数，然后计算散列地址

折叠法

把关键字自左到右分成位数相等的几部分，每一部分的位数应与散列表地址位数相同，把这些部分的数据叠加起来得到散列地址。叠加可以移位或者折叠

冲突解决方法

  1. 开放定址法

    当冲突发生时，为key生成一个探查地址序列，当冲突时依次探查。

    1. 线性探查法：继续向相邻的下一个位置探查
    2. 二次探查法：d, d+1^2, d-1^2,d+2^2,d-2^2,…
    3. 伪随机探查法：使用随机数生成器来生成随机的探查地址序列
    4. 双散列探查法：Hash(key) + i * ReHash(key)

  2. 链接法

    散列表中的每个地址都是一个链表的表头，直接将相同地址的记录加入链表中

  3. 桶定址法

    把记录分为若干存储桶，每个存储通包含一个或多个存储位置，一个存储桶内的各存储位置用指针连接起来。散列函数f将关键字key映射到f(key)号存储桶。如果某个地址关联的桶满了，则可以使用开放定址法解决冲突。

排序

内部排序

插入排序

1. 直接插入排序

  • 空间复杂度：O(1)

  • 时间复杂度：

    • 最好：O(n)
    • 最坏：O(n^2)
    • 平均：O(n^2)
  • 稳定性：稳定

2.折半插入排序

  • 空间复杂度：O(1)

  • 时间复杂度：

    • 最好：O(n)
    • 最坏：O(n^2)
    • 平均：O(n^2)

  • 稳定性：稳定

  • 特点：对前p个有序序列进行二分查找；减少了比较次数，并未减少移动次数

3. 希尔排序

  • 空间复杂度：O(1)

  • 时间复杂度：

    • 最好：O(n)
    • 最坏：
    • 平均：O(n^1.3)

  • 稳定性：不稳定

  • 特点：时间复杂度与增量的选择有关

交换排序

1. 冒泡排序

  • 空间复杂度：O(1)

  • 时间复杂度：

    • 最好：O(n)
    • 最坏：O(n^2)
    • 平均：O(n^2)
  • 稳定性：稳定

2. 快速排序

  • 空间复杂度：O(logn)~O(n)

  • 时间复杂度：

    • 最好：O(nlogn)
    • 最坏：O(n^2)
    • 平均：O(nlogn)

  • 稳定性：不稳定

  • 特点：基于分治法；递归地将比轴元素小的放在左边、大的放在右边，最后直接合并

选择排序

1. 简单选择排序

  • 空间复杂度：O(1)

  • 时间复杂度：

    • 最好：
    • 最坏：
    • 平均：O(n^2)

  • 稳定性：不稳定

  • 特点：每次选取最小（最大）的记录与第p个记录进行交换

2. 堆排序

  • 空间复杂度：O(1)

  • 时间复杂度：

    • 最好：
    • 最坏：O(nlogn)
    • 平均：O(nlogn)
  • 稳定性：不稳定
  • 特点：由于存在一个O(n)的建堆过程，适用于数据量较大的情况

归并排序

  • 空间复杂度：O(n)

  • 时间复杂度：

    • 最好：
    • 最坏：O(nlogn)
    • 平均：O(nlogn)

  • 稳定性：稳定

  • 特点：算法复杂度较低，先进行划分，在合并时排序

基数排序

借助多关键字排序的思想对单逻辑关键字进行排序的方法。

  1. 高位优先法（MSDF）

  2. 低位优先法（LSDF）

  • 空间复杂度：O(n)

  • 时间复杂度：（d为关键字的个数）

    • 最好：O(dn)
    • 最坏：O(dn)
    • 平均：O(dn)
  • 稳定性：稳定

排序方法的选择

  1. 元素个数较多，排序关键字随机分布且对稳定性不做要求时：快速排序
  2. 元素个数较多，内存空间允许，要求排序稳定时：二路归并
  3. 元素个数较大，关键字可能出现正序或逆序的情形，对稳定性不做要求：堆排序或二路归并
  4. 元素个数较小，要求稳定：直接插入排序
  5. 元素个数较小，稳定性无要求：直接选择排序

外部排序

分为两个阶段：预处理和归并处理

预处理：

  1. 初始化堆
    1. 从磁盘读入M个记录放到内存数组中
    2. 设置堆末尾last = M-1
    3. 建立最小堆
  2. 重复一下步骤直到堆为空，即last = -1
    1. 把根结点送出到输出缓冲区
    2. 设R是输入缓冲区的下一条记录，如果R大于刚刚输出的关键字，则将其放在根结点；否则放在last位置，同时last–
    3. 重新调整堆

多路归并：

赢者树

败者树

每个叶子结点是某一个串的当前处理的数据，当从小到大排序时，赢者树将左右子结点的最小值所在顺串的索引放在内部结点中（而败者树是将最大值放在结点中）,同时二者均向上推举最小值进行下一轮比较，最后赢者树的根结点即为当前树中最小值的索引，而败者树会多一次推举。

败者树的优点在于简化了重构，不需要与兄弟结点的记录进行比较。

对m个串使用k路赢者树或者败者树进行归并时时间复杂度为O(k+nlogk)
